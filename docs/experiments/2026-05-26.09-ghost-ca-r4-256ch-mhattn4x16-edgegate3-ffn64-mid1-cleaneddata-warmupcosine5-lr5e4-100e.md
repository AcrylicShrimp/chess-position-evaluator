## Goal

Retrain the stacked edge-gate FFN architecture on the current cleaned processed dataset using the same architecture and training recipe as the prior strongest stacked edge-gate FFN run.

This experiment is meant to isolate whether removing a low-rate set of corrupted or extreme material positions from the processed dataset creates a measurable improvement for the same model family and recipe.

## Hypothesis Or Expectation

The cleaned-data retrain should reproduce the prior same-recipe stacked edge-gate FFN run. If the removed garbage data mattered despite its low rate, the full validation and test BCE should improve against that exact historical run.

Expected outcomes:

- The primary comparison is the identical pre-cleaning stacked edge-gate FFN run at `docs/experiments/2026-05-26.02-ghost-ca-r4-256ch-mhattn4x16-edgegate3-ffn64-mid1-warmupcosine5-lr5e4-100e.md`.
- The cleaned-data retrain should at least reproduce the prior full validation BCE `0.6010095617921948` and full test BCE `0.6010760372153482`.
- A meaningful cleanup signal requires the final full validation and test metrics to beat the same-recipe reference, not merely to beat weaker or structurally different architecture experiments.
- If the result is statistically indistinguishable from the prior run, the cleaned rows were probably too rare to affect this recipe materially, even though keeping the dataset clean remains the correct hygiene policy.

## Key Characteristics

- Model variant: `stacked-edge-gate-ffn`.
- Architecture: `ValueOnlyModel` with `CHANNELS=256`.
- Attention stack: `3` edge-gated `BoardAttentionBlock` layers with `4` heads, `16` head dimension, and FFN hidden width `64`.
- Attention insertion point: after trunk block 3.
- Scheduler: warmup-cosine, 5 warmup epochs, no warm restart.
- Training recipe intentionally follows `docs/experiments/2026-05-26.02-ghost-ca-r4-256ch-mhattn4x16-edgegate3-ffn64-mid1-warmupcosine5-lr5e4-100e.md`.

## Verdict Rationale

Pass for the positive cleanup-signal criterion: both full validation and full test BCE beat the same-recipe historical reference.

The effect is small, not strong by the pre-recorded interpretation rule. Full validation improved by `0.00016170047729702208` BCE and full test improved by `0.00015496495275379996` BCE, both below the stronger `~0.0002` full-test improvement threshold. The sampled best validation loss also remained slightly worse than the historical sampled best (`0.6006121728934494` vs `0.6004527146909752`), so the safest conclusion is that cleaning the low-rate garbage rows produced a consistent but modest positive effect.

ECE regressed on both splits, so calibration should not be claimed to have improved even though BCE and CP-equivalent MAE improved.

## Issues And Anomalies

- This is a cleaned-data retrain, not a new architecture. Any result changes should be interpreted against the same-recipe historical stacked architecture reference.
- The sampled best validation loss did not beat the historical sampled best, even though full validation and test BCE did beat their historical full-eval references.
- ECE worsened on both validation and test splits, despite BCE and CP-equivalent MAE improvements.

## Follow-Up

- Treat the cleaned dataset as the default substrate for future architecture experiments, because the same-recipe retrain produced a consistent positive BCE signal.
- Avoid claiming a large cleaning effect from this single run; repeat this exact recipe only if variance quantification becomes important.
- Investigate why ECE worsened despite BCE and CP-equivalent MAE improving.
