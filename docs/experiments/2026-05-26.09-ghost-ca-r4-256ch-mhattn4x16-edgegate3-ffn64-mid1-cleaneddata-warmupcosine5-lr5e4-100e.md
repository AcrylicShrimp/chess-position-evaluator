## Goal

Retrain the stacked edge-gate FFN architecture on the current cleaned processed dataset.

This experiment is meant to decide whether the previously strongest stacked edge-gate FFN baseline remains the best architecture after the dataset cleanup and after the parallel-fusion material-features run showed a meaningful but less efficient result.

## Hypothesis Or Expectation

The stacked edge-gate FFN architecture should remain the primary baseline because it previously achieved the best full test BCE while being smaller and faster than the parallel local/global fusion model.

Expected outcomes:

- The cleaned-data retrain should beat the parallel-fusion material-features full test BCE `0.602763`.
- The cleaned-data retrain should stay close to the historical stacked edge-gate FFN full test BCE `0.6010760372153482`.
- If the result degrades substantially, the cleaned data may have changed the model ranking and all current architecture comparisons should be re-centered around cleaned-data retrains.

## Key Characteristics

- Model variant: `stacked-edge-gate-ffn`.
- Architecture: `ValueOnlyModel` with `CHANNELS=256`.
- Attention stack: `3` edge-gated `BoardAttentionBlock` layers with `4` heads, `16` head dimension, and FFN hidden width `64`.
- Attention insertion point: after trunk block 3.
- Scheduler: warmup-cosine, 5 warmup epochs, no warm restart.
- Training recipe intentionally follows `docs/experiments/2026-05-26.02-ghost-ca-r4-256ch-mhattn4x16-edgegate3-ffn64-mid1-warmupcosine5-lr5e4-100e.md`.
- Dataset focus: current cleaned `data/processed/*.chesseval` artifacts.
- Tracking TODO: `docs/todos/2026-05-26.01.retrain-stacked-edge-gate-ffn-on-cleaned-data.md`.
