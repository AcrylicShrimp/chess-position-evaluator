## Goal

Train and evaluate a funnel CNN plus narrow deep attention model.

This experiment tests whether a wide local CNN encoder followed by channel compression and six narrow edge-gated attention blocks is a better parameter-budget tradeoff than the current `C256` stacked edge-gate FFN baseline.

## Hypothesis Or Expectation

The model may improve chess evaluation by spending less budget on wide `C256` channel projections and more budget on repeated global relation updates at `C128`.

The main risk is that compressing to `C128` before the attention stack may discard local tactical features, or that six sequential attention blocks may be slower than the accuracy gain justifies.

## Key Characteristics

- Model variant: `funnel-cnn224-160-128-attn6-edgegate`.
- Architecture: `24 -> 224`, two `C224` Ghost blocks, `224 -> 160`, two `C160` Ghost blocks, `160 -> 128`, two `C128` Ghost blocks, six `C128` q+k edge-gated attention blocks.
- Attention: `4` heads, `16` head dim, FFN hidden size `64`.
- Value head: material-difference scalar semantics matching the stacked baseline.
- No CNN/attention interleaving in this first funnel experiment.
- Design doc: `docs/designs/2026-05-27.01-funnel-narrow-attention.md`.

Interpretation:

- There is no simple channel-rank collapse in the funnel model. Every funnel stage is full-rank under the original 1% threshold.
- The sharper signal is spectral concentration. The final funnel attention block drops from `99/128` to `21/128` directions above 10% of the top singular value, while the stacked baseline final block has `167/256`.
- The `C128` bottleneck is therefore not obviously dead, but the last narrow attention block appears to compress useful variation into a much smaller number of dominant axes before the value head.
- The value-head evidence projection is also less balanced in the funnel model, which is consistent with a narrower final representation feeding the scalar evaluator.

## Verdict Rationale

Fail. The funnel CNN plus narrow deep attention model trained stably and was faster than the stacked reference in compiled forward timing, but it did not match the cleaned stacked edge-gate FFN baseline on sampled validation or full validation/test evaluation.

The final full test BCE regressed by `+0.0012348850717816`, and full validation BCE regressed by `+0.0011785552424321`. ECE and CP-equivalent MAE also regressed on both splits.

The main architectural conclusion is that aggressive compression to `C128` before six attention blocks did not preserve enough representational efficiency to beat the `C256` stacked edge-gate FFN baseline, even though the compiled execution path was attractive.

The follow-up rank analysis refines this conclusion: the compression path does not simply collapse rank, but the final narrow attention block becomes substantially more top-heavy than the stacked baseline. The likely failure mode is not unused channels; it is late-stage spectral concentration before the value head.

## Issues And Anomalies

- `max-autotune` has a high first-step compile cost. It is only slightly faster than `reduce-overhead` in steady-state train-step timing, so the full run should use it mainly because this experiment has enough steps to amortize the compile cost.
- The sampled validation curve stayed roughly `0.001` BCE behind the cleaned stacked edge-gate FFN baseline from epoch 60 onward.
- Full ECE regressed despite stable training, so calibration should not be claimed to have improved.
- Rank analysis found a large late-stage spectrum shift in the funnel model: `attention_block_5` had only `21/128` directions above 10% of the top singular value, compared with `99/128` in `attention_block_4`.
- Implementation and initial experiment log were committed as `f4cbde3`.

## Follow-Up

- Do not treat `funnel-cnn224-160-128-attn6-edgegate` as the new baseline.
- Consider a less aggressive compression ablation, such as keeping attention at `C160` or `C192`, if continuing the funnel idea.
- Consider a conservative learning-rate ablation (`7.5e-4`) only after deciding whether the architecture is worth more tuning.
- If testing a higher LR, treat it as an optimizer ablation, not as proof that the funnel architecture is fixed; rank evidence points to final-block representation concentration.
- Consider a variant that removes the sixth attention block or widens only the final attention/value-head interface to test whether the last-block spectrum concentration is causal.
- Keep the compiled benchmark result: this architecture is useful evidence that narrower attention can be faster under `torch.compile`, even when accuracy regresses.
