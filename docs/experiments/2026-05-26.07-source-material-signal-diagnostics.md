## Goal

Measure whether side-to-move-relative material difference and absolute material difference are useful signals for the engine target in the staging/source data.

This experiment is meant to replace misleading prior-only BCE interpretation with direct source-level diagnostics: `material_diff`, `abs(material_diff)`, `relative_cp`, `abs(relative_cp)`, and `relative_cp - 100 * material_diff`.

## Hypothesis Or Expectation

Signed material difference should correlate positively with side-to-move-relative engine cp. Absolute material difference should correlate positively with absolute cp magnitude and decisive-position rates, but it should not by itself provide direction.

## Key Characteristics

- No model checkpoint is loaded.
- Analysis reads staging DuckDB rows rather than processed `.chesseval` labels.
- Target is engine `cp` from the Lichess eval source after mate values have already been clamped to `+/-2000` during staging creation.
- This does not measure real game outcome win rate because the current source data does not include game results.
- The diagnostic recomputes strict standard-position filters before accumulating material statistics.
- Planned report includes signed material buckets, absolute material buckets, residual cp buckets, threshold rates, sign agreement, and correlations.

## Verdict Rationale

This is informational because it measures source-level signal structure rather than testing a model-change pass/fail criterion.

The result supports keeping material information as a meaningful model input. Signed material difference has a moderate positive correlation with side-to-move-relative cp, and the material/cp sign agrees in about `73.3%` of non-neutral nonzero-material rows. Absolute material difference also has a moderate positive correlation with absolute cp magnitude and sharply increases decisive-position rates.

The result also reinforces that fixed prior BCE is not the right truth test for material value. Material buckets have wide cp distributions and many neutral or tactical-compensation positions. For example, `material_diff=+1` averages about `+138cp`, but its median cp is only `+7cp`; a fixed probability prediction cannot represent that mixture.

## Issues And Anomalies

- Prior-only BCE is not a reliable test of whether `100cp` per material point is a useful material heuristic, because material buckets contain wide and non-linear cp mixtures.
- The source target is engine cp, not actual game outcome win rate.
- `uv add duckdb` emitted an unrelated warning that `numpy==2.4.0` is yanked for a backward compatibility bug. The command still resolved and installed `duckdb==1.5.3`.
- The 1M sample is a deterministic prefix from the shuffled staging table, not a full-source sweep.
