## Goal

Test whether replacing static chess-geometry logit bias with a content-conditioned edge gate improves the current best 256-channel, 4-head board attention model.

This experiment should inform whether relation and distance metadata are more useful as dynamic query/key routing signals than as unconditional logit constants.

## Hypothesis Or Expectation

Static relation and distance logits may be too blunt for chess because the usefulness of a rank, file, diagonal, knight, pawn, or distance relation depends heavily on the current position.

The edge gate may improve performance by letting the current query and key features decide whether a relation or distance bucket should increase or decrease attention for a specific square pair.

Primary pass criterion:

- Full validation BCE must be below the static relation-bias run: `0.6075506562081671`.
- Full test BCE must be below the static relation-bias run: `0.6075916815231461`.

Secondary reference:

- Full validation BCE should also be compared against the no-static 4-head attention run: `0.607886908`.
- Full test BCE should also be compared against the no-static 4-head attention run: `0.607948302`.

If validation and test disagree, treat the result as inconclusive and prefer replication.

## Key Characteristics

- Architecture variant: `ValueOnlyModel` with `CHANNELS=256`.
- Trunk: six `GhostShuffleBlock` modules with coordinate attention.
- Board attention: one `NaiveBoardSelfAttention` module after the third trunk block.
- Attention layout: `4` heads, `16` dimensions per head, total projection dimension `64`.
- Attention scaling: per-head scale `16 ** -0.5`.
- Removed static geometry logits:
  - no `rel_bias`
  - no `dist_bias`
- New edge gate parameters:
  - `rel_gate_q`: zero-initialized `[4, 9, 16]`
  - `rel_gate_k`: zero-initialized `[4, 9, 16]`
  - `dist_gate_q`: zero-initialized `[4, 8, 16]`
  - `dist_gate_k`: zero-initialized `[4, 8, 16]`
- Edge-gate parameters: `2,176`.
- Total parameters: `223,573`.
- Direct comparison target: `docs/experiments/2026-05-25.05-ghost-ca-r4-256ch-mhattn4x16-rel9dist8-mid1-bestrecipe-100e.md`.
- Design doc: `docs/designs/2026-05-26.00-attention-edge-gate.md`.

The learned edge gate is not inert: it produces non-trivial additive logits, often around half the mean magnitude of the base query-key logits and occasionally much larger. The learned activation is mostly negative on the inspected validation batch, with strongest average magnitudes on pawn-attack geometry, same-square, anti-diagonal, and king-adjacent pairs.

## Verdict Rationale

Verdict is `Fail` against the primary pass criterion because full validation and full test BCE both missed the static relation-bias targets.

The result is still informative:

- Edge gating beat the no-static 4-head attention run on full validation and test BCE.
- Edge gating did not beat the static relation-bias run on BCE.
- Edge gating produced worse test ECE than both static relation-bias and no-static 4-head attention, contradicting the expectation that content-conditioned relation routing would improve calibration.
- Sampled validation looked better than full validation at the end, so future decisions should rely on the full-split metrics.

## Issues And Anomalies

- The initial loop-based edge-gate implementation was too slow for a full experiment, stabilizing around only a few steps per second in short smoke runs.
- The implementation was vectorized into precomputed per-pair edge query/key vectors before full training.
- The vectorized 128-step smoke stabilized near `12.2` steps/s after compile warmup, which is slower than the previous attention model but feasible for this experiment.
- Full training throughput was about `32` steps/s after compile warmup with training workers enabled.
- Sampled validation set a new best at epoch 100, but full validation did not beat the static relation-bias target.
- Calibration worsened on the full test split despite the content-conditioned gate design.

## Follow-Up

- Use the gate inspection results to decide whether a constrained dynamic gate is worth testing.
- Consider testing a cheaper or more regularized dynamic gate, such as query-only, key-only, per-head scalar gates, or bounded/tanh-gated logits.
- Treat static relation-bias as the current best 4-head geometry variant until a follow-up dynamic design beats it on full validation and test.
