## Goal

Train a k-only edge-gated parallel CNN/attention model while keeping the previous wide `1x1` fusion path and no material-score inputs.

This experiment isolates whether the previous k-only late-evidence failure was caused primarily by the narrow late-evidence bottleneck or by removing q-side edge gate information.

## Hypothesis Or Expectation

If the late-evidence bottleneck caused most of the regression, this k-only wide-fusion run should recover close to the previous no-material parallel fusion result.

If q-side edge gate information is important, this run should remain meaningfully worse than the previous q+k no-material wide-fusion run despite restoring the wide fusion path.

## Key Characteristics

- Model variant: `parallel-cnn-attn-kedge-fuse-no-material`.
- Shared encoder: `GhostShuffleBlock x3`.
- Local branch: `GhostShuffleBlock x3`.
- Global branch: `BoardAttentionStack x3`, each with 4 heads, 16 head dim, FFN hidden size 64, and k-only dynamic edge gates.
- Fusion: concat local/global features followed by `Conv2d 1x1 512 -> 256`, BatchNorm, Hardswish.
- Value head: `ResidualValueHead`.
- Explicit material-score input: removed.
- Additive material prior: not used.
- Dataset: strict-filtered processed data under `data/processed`.
- Scheduler recipe: warmup-cosine, 5 warmup epochs, no warm restart.

## Verdict Rationale

Fail against the predeclared success criteria.

The k-only wide-fusion model fixed the severe k-only late-evidence regression and was faster than the q+k wide-fusion reference in the pre-training forward benchmark. However, it did not match the previous q+k no-material wide-fusion model on full validation or full test BCE.

The full test BCE was worse by about `0.00079`, exceeding the `0.0004` inconclusive band. Test ECE also regressed slightly, from `0.0005251688410744738` to `0.0006240813833717768`.

Interpretation: the narrow late-evidence bottleneck was a major problem in the prior k-only run, but q-side edge-gate information still appears to provide a small, persistent accuracy benefit when wide fusion is retained.

## Issues And Anomalies

- The exact containing commit hash is available from Git history; it is not embedded in this file to avoid self-referential hash churn when amending.
- Sampled validation briefly led the q+k reference at epoch 10, then became unstable around epoch 20 before recovering. The final full evaluation still favored q+k.
- No smoke, training, checkpointing, or full-evaluation crash was observed.

## Follow-Up

- Prefer q+k wide fusion over k-only wide fusion when optimizing for best accuracy.
- Keep k-only wide fusion as a speed/accuracy tradeoff candidate if inference speed becomes a priority.
- Do not pursue the late-evidence bottleneck design without a stronger fusion contract; this run suggests the bottleneck, not k-only alone, caused most of that earlier failure.
- If continuing the q-only/k-only split, run the symmetric q-only wide-fusion ablation before making a final statement about which side of the edge gate matters more.
