## Goal

Test whether adding one simple global square-to-square attention mixer to the existing value-model trunk improves dataset-level value prediction over the 90/5/5 split baseline.

This experiment is meant to decide whether naive board self-attention is worth keeping as a serious follow-up direction before trying richer Transformer-style blocks, LayerNorm, multi-head variants, or local/global fusion designs.

## Hypothesis Or Expectation

Chess positions contain long-range interactions that are awkward for a purely local CNN trunk. A single full-board self-attention mixer should improve validation and test BCE if direct square-to-square message passing is useful.

Pass criterion:

- Full validation BCE and full test BCE should both beat the baseline checkpoint.
- A full test BCE improvement of roughly `0.001` or more is treated as a meaningful positive signal for this first experiment.

## Key Characteristics

- Architecture variant: one `NaiveBoardSelfAttention(channels=256, attn_dim=64)` module.
- Placement: after the first three `GhostShuffleBlock` layers and before the remaining three.
- Trunk width unchanged: `CHANNELS=256`.
- Ghost block count unchanged: `BLOCKS=6`.

## Verdict Rationale

Verdict is `Pass` because the attention model beats the baseline on both full validation and full test BCE, and the full test BCE improvement is approximately `0.00160`, above the planned `~0.001` threshold for a meaningful first signal.

The result supports keeping naive board self-attention as an active research direction. It does not prove that attention specifically caused the gain, because the model also gained `65,536` parameters.

## Issues And Anomalies

- The implementation was trained before being committed, so the exact source state is represented by the current working tree plus base commit `1055e36` until a follow-up commit is made.
- The best checkpoint's epoch field is `69`, while the final checkpoint's epoch field is `99`; this reflects existing checkpoint-save semantics where the final checkpoint stores the zero-based loop index and the best checkpoint stores `epoch + 1`.
- The attention run improved strongly through epoch 69, then plateaued and partially rebounded before ending at validation `0.6087`.
- Capacity is a confound: the attention branch increases parameter count from `155,861` to `221,397`.

## Follow-Up

- Consider a parameter-matched control model to separate global attention benefit from capacity increase.
