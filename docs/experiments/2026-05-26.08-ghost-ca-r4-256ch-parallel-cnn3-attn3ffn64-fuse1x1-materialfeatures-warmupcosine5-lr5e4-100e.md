## Goal

Retry the parallel CNN attention fusion experiment after replacing the hard additive material prior with material input features.

This experiment is meant to decide whether the previous parallel-fusion failure was partly caused by forcing `material_prior_logit + residual_logit`, rather than letting the value head learn when to trust material information.

## Hypothesis Or Expectation

Material difference is a useful source-level signal, but prior-only BCE was a misleading diagnostic because each material bucket has a wide cp mixture. Feeding material-derived features to the value head should preserve useful material information while avoiding the optimization burden of canceling an additive fixed prior.

Expected improvement signals:

- Lower gradient-norm tail than the hard-prior parallel run.
- Better early sampled validation loss than the hard-prior parallel run.
- Full validation/test BCE closer to or better than the stacked edge-gate FFN warmup-cosine reference.

## Key Characteristics

- Model variant: `parallel-cnn-attn-fuse`.
- Shared encoder: `GhostShuffleBlock x3`.
- Local branch: `GhostShuffleBlock x3`.
- Global branch: `BoardAttentionStack x3`, each with 4 heads, 16 head dim, and FFN hidden size 64.
- Fusion: concatenate local and global branch outputs, then `Conv2d 1x1 512 -> 256`, BatchNorm, Hardswish.
- Output contract: `value_logit = value_head(fused_features, material_features)`.
- No additive material prior is added to the output logit.
- Material features passed to the value head:
  - `tanh(material_diff / 5)`
  - `tanh(abs(material_diff) / 5)`
  - `sigmoid(material_diff * log(10) / 4)`
- Scheduler recipe: warmup-cosine, 5 warmup epochs, no warm restart.
- Motivation source: `docs/experiments/2026-05-26.07-source-material-signal-diagnostics.md`.
- Historical failed reference: `docs/experiments/2026-05-26.05-ghost-ca-r4-256ch-parallel-cnn3-attn3ffn64-fuse1x1-materialprior-warmupcosine5-lr5e4-100e.md`.

## Verdict Rationale

Promising. The final full test BCE `0.602763` falls inside the pre-registered `Promising` band and clearly beats the hard-prior parallel-fusion reference `0.6040597092544988`. This strongly supports the hypothesis that the previous parallel-fusion result was harmed by forcing an additive fixed material prior.

The result does not beat the smaller and faster stacked edge-gate FFN reference: test BCE is worse by about `0.001687`, and ECE is worse by about `0.000687`. The architecture is therefore not the new primary baseline on efficiency or final score, but it is a meaningful architecture signal rather than a failed run.

## Issues And Anomalies

- This is not a pure branch-topology retry: the value-head contract changes from additive prior residual learning to material-conditioned prediction.
- Existing hard-prior parallel checkpoints are not expected to load into the revised value head without retraining.
- The current model is larger and slower than the stacked edge-gate FFN reference: `589397` parameters versus `457685`, and observed training throughput is about `26` steps/s versus the stacked edge-gate FFN run's documented `~32` steps/s.
- At epoch 30, the current best sampled validation loss `0.6096741808395759` still trails the stacked edge-gate FFN epoch 30 reference `~0.6078`, despite the larger parameter count.
- At epoch 50, the current best sampled validation loss `0.6056829515625449` trails the stacked edge-gate FFN epoch 50 reference `~0.6040` by about `0.0017`, so the gap is narrowing but has not closed.
- At epoch 60, the latest best sampled validation loss `0.6047778789903603` still trails the stacked edge-gate FFN epoch 60 reference `~0.6026` by about `0.0022`.
- At epoch 70, the latest best sampled validation loss `0.6036351971766528` still trails the stacked edge-gate FFN epoch 70 reference `~0.6017` by about `0.0019`, but it has moved past the old hard-prior parallel full-test reference.
- At epoch 80, the latest best sampled validation loss `0.602882818848479` entered the pre-registered `Promising` band, while still trailing the stacked edge-gate FFN epoch 80 reference `~0.6013` by about `0.0016`.
- At epoch 90, the latest best sampled validation loss `0.6024887024187574` remained in the `Promising` band, while still trailing the stacked edge-gate FFN epoch 90 reference `~0.6005` by about `0.0020`.
- Final full evaluation showed better BCE than hard-prior parallel fusion but worse ECE than both the hard-prior parallel fusion and stacked edge-gate FFN references.
