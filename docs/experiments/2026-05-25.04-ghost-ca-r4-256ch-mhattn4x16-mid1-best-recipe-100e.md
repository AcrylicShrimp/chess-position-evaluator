## Goal

Test whether replacing the current best 256-channel single-head board attention layout with a 4-head layout improves or hurts performance when every other known best setting is held constant.

This experiment is meant to isolate the effect of multi-head attention at fixed trunk width, fixed attention projection width, fixed parameter count, fixed placement, and fixed training recipe.

## Hypothesis Or Expectation

Multi-head attention may improve square-to-square interaction modeling by letting four smaller attention maps specialize independently. The risk is that splitting the same `64` attention dimensions into `4 x 16` heads may reduce per-head expressiveness and underperform the single-head `64`-dimensional attention mixer.

Pass criterion:

- Full validation BCE and full test BCE should both beat the 256-channel single-head attention run.
- If both full-split BCE metrics are worse than the 256-channel single-head attention run, treat multi-head as not helpful under this setup.
- If validation and test disagree, treat the result as inconclusive and prefer follow-up replication.

## Key Characteristics

- Architecture variant: `ValueOnlyModel` with `CHANNELS=256`.
- Trunk: six `GhostShuffleBlock` modules with coordinate attention.
- Board attention: one `NaiveBoardSelfAttention` module after the third trunk block.
- Attention layout: `4` heads, `16` dimensions per head, total projection dimension `64`.
- Attention scaling: per-head scale `16 ** -0.5`.
- Total parameters: `221,397`.
- Attention parameters: `65,536`.
- Direct comparison target: `docs/experiments/2026-05-25.01-naive-board-self-attention-100e.md`.
- The comparison target used `CHANNELS=256`, attention projection dimension `64`, no multi-head split, and total parameters `221,397`.
