## Goal

Run a 40-epoch gate for the depth-preserving refresh architecture with double the original base learning rate.

This experiment tests whether the `funnel-cnn224-160-128-attn6-refresh3-edgegate` model is learning-rate limited upward. The `lr=0.00025` run underfit badly under the same 100-epoch budget, so this run checks the opposite direction from the `lr=0.0005` reference.

## Hypothesis Or Expectation

Increasing the base learning rate to `0.001` may accelerate fitting and improve the early validation curve if the model is under-optimized at lower rates.

Expected positive evidence:

- sampled validation at epoch 40 at least competitive with the `lr=0.0005` gate value `0.6062`
- no NaN, divergence, or sustained validation degradation after warmup
- no severe grad-norm instability beyond what grad clipping can contain

## Key Characteristics

- Model variant: `funnel-cnn224-160-128-attn6-refresh3-edgegate`.
- Architecture is unchanged from the depth-refresh experiments.
- Training recipe is unchanged except `lr=0.001`.
- `eta_min` is intentionally kept at `0.000001` to isolate the base-LR change.
- This is a 40-epoch gate run, not a full promotion run.
- If the gate is promising, the follow-up is a 100-epoch full run with the same LR.

## Verdict Rationale

Fail. Doubling the base learning rate produced a fast early curve but then caused clear optimizer instability.

The run looked promising through epoch 21, beating nearby lower-LR trajectories at the same epoch range. However, validation jumped from `0.6092` to `0.6833` at epoch 22, training loss also jumped, and W&B showed extreme grad-norm spikes. This violates the gate stability requirement, so `lr=0.001` is too high for this architecture and schedule.

## Issues And Anomalies

- Severe grad-norm explosion after epoch 21.
- W&B still reported the run as `running` after process termination because the process group was killed during the failed gate.
- No full validation/test evaluation was run; the instability alone was sufficient to reject the gate.

## Follow-Up

- Treat `0.001` as too high for the current architecture and warmup-cosine schedule.
- Bracket the useful range between `0.0005` and `0.001`; a `0.00075` gate is the most direct next LR ablation.
- Consider schedule/optimizer stability separately if higher LR remains attractive, because this run showed good early speed before collapse.
