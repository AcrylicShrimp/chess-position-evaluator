## Goal

Repeat the depth-preserving refresh architecture with half the previous base learning rate.

This experiment tests whether the `funnel-cnn224-160-128-attn6-refresh3-edgegate` model needs a lower learning-rate scale to train cleanly. The previous `lr=0.0005` run improved steadily only after the cosine schedule cooled, but still failed full validation/test BCE against the previous six-attention funnel and the cleaned stacked baseline.

## Hypothesis Or Expectation

Lowering the base learning rate to `0.00025` may reduce early high-LR drag for the deeper attention-refresh stack and produce better full validation/test BCE.

Expected positive evidence:

- less noisy sampled validation before epoch 50
- full validation/test BCE better than the `lr=0.0005` depth-refresh run
- ideally full test BCE at least competitive with the previous six-attention funnel `0.602155957334376`

## Key Characteristics

- Model variant: `funnel-cnn224-160-128-attn6-refresh3-edgegate`.
- Architecture is unchanged from the previous depth-refresh experiment.
- Training recipe is unchanged except `lr=0.00025`.
- `eta_min` is intentionally kept at `0.000001` to isolate the base-LR change.
- This is an optimization ablation, not a new architecture.

## Verdict Rationale

Fail. The lower-learning-rate run underfit relative to the `lr=0.0005` reference.

The sampled validation curve stayed about `0.002` BCE behind the reference through the final third of training, and full validation BCE was `0.605336` versus `0.6031740756086265` for the `lr=0.0005` run. This rejects the hypothesis that halving the base learning rate improves this architecture under the current 100-epoch budget.

## Issues And Anomalies

- Lowering the base learning rate made the run substantially slower to fit under the same epoch budget.
- Full test and rank analysis were skipped because full validation already answered the optimization question.

## Follow-Up

- Run the opposite learning-rate ablation with `lr=0.001` as a 40-epoch gate to test whether this architecture is actually learning-rate limited upward.
- Watch grad norm closely because the doubled base learning rate may reintroduce instability even with warmup and clipping.
