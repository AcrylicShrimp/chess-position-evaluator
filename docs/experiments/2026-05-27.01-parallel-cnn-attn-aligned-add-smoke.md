## Goal

Verify that the `parallel-cnn-attn-aligned-add` variant can be constructed, exposed through the CLI, pass unit contracts, and run a short training smoke before any full comparison run is started.

This experiment informs whether option 1 from the principled local/global fusion design is safe to take into a full training run.

## Hypothesis Or Expectation

The aligned-add variant should preserve the same board/value contracts as `parallel-cnn-attn-fuse`, add only a small parameter increase, and complete a target-batch smoke run without shape, checkpoint, scheduler, or CUDA errors.

## Key Characteristics

- Architecture: shared 3-block CNN trunk, local 3-block CNN branch, global 3-layer attention+FFN branch.
- Fusion change: replace concat+1x1 fuse with per-branch BatchNorm, per-branch 1x1 projection, elementwise add, final BatchNorm, and Hardswish.
- Model variant: `parallel-cnn-attn-aligned-add`.
- Value head: `MaterialFeatureValueHead`, matching the current material-feature parallel fusion path.
- Dataset: regenerated strict-filtered processed data under `data/processed`.
- Training regime: short offline W&B smoke using the cleaned-data warmup-cosine recipe.

## Procedure

```bash
uv run python -m unittest python.tests.test_model_attention python.tests.test_model_material python.tests.test_cli_smoke -v
uv run python -m unittest discover -s python/tests -t . -v
git diff --check
```

Smoke training command:

```bash
WANDB_API_KEY=offline WANDB_PROJECT=chess-position-evaluator WANDB_MODE=offline PYTHONUNBUFFERED=1 uv run cpe train ghost-ca-r4-256ch-parallel-cnn3-attn3ffn64-alignedadd-materialfeatures-warmupcosine5-lr5e4-smoke-20260527-1252 \
  --model-variant parallel-cnn-attn-aligned-add \
  --epochs 1 \
  --steps 128 \
  --batch 2048 \
  --lr 0.0005 \
  --wd 0.0001 \
  --scheduler warmup-cosine \
  --warmup-epochs 1 \
  --warmup-start-factor 0.1 \
  --eta-min 1e-6 \
  --grad-clip 5 \
  --train-workers 4 \
  --val-workers 2 \
  --no-upload-checkpoints
```
