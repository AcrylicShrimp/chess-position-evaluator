## Goal

Run a gated depth-preserving refresh probe for the funnel attention family.

This experiment tests whether adding CNN refresh blocks after pairs of narrow attention blocks can preserve the previous six-attention funnel's relation-mixing depth while reducing the final-stage spectral concentration observed in that run.

The decision this experiment informs is whether the funnel-refresh direction deserves a full 100-epoch run or should be deprioritized in favor of the cleaned stacked baseline family.

## Hypothesis Or Expectation

The expected positive signal is that preserving six attention blocks while adding three refresh blocks improves sampled validation loss versus the previous six-attention funnel by epoch 40 and avoids the severe final-attention spectrum concentration.

The main risk is that the prior interleaved rank improvement came from added CNN capacity rather than useful task signal, or that `C128` remains too narrow regardless of refresh.

## Key Characteristics

- Model variant: `funnel-cnn224-160-128-attn6-refresh3-edgegate`.
- Architecture: `24 -> 224`, two `C224` Ghost blocks, `224 -> 160`, two `C160` Ghost blocks, `160 -> 128`, two `C128` Ghost blocks, then `[C128 q+k edge-gated attention block -> C128 q+k edge-gated attention block -> C128 Ghost refresh] x3`.
- Attention: `4` heads, `16` head dim, FFN hidden size `64`.
- Value head: material-difference scalar semantics matching the cleaned stacked baseline and previous funnel experiments.
- Distinct from the previous six-attention funnel: adds three CNN refresh blocks while preserving six attention blocks.
- Distinct from the previous interleaved funnel: preserves six attention blocks instead of reducing attention depth to three.
- Design doc: `docs/designs/2026-05-28.00-funnel-depth-preserving-refresh.md`.

## Verdict Rationale

Pass for the pre-registered gate. The epoch-40 sampled validation loss was `0.6062`, better than the previous six-attention funnel's `0.6072` and within the strong extension threshold `<= 0.6067`. Rank analysis also avoided the previous final-attention concentration failure: final attention `rank>10%` improved from `21/128` to `48/128`, and the final refresh block measured `86/128`.

The full 100-epoch result is still pending. Because the existing resume path restores the checkpoint scheduler horizon, extending the 40-epoch run directly would not produce a clean 100-epoch warmup-cosine schedule. The full run should start fresh with `--epochs 100`.
