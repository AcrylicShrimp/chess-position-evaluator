## Goal

Evaluate whether a parallel local/global value trunk improves chess position evaluation compared with the current stacked edge-gated attention family.

This experiment is meant to decide whether separating local CNN processing and global attention processing before late fusion is a better architecture direction than inserting attention into a single sequential CNN trunk.

## Hypothesis Or Expectation

The parallel local/global split should improve full test BCE if the attention branch learns useful long-range interactions that the local CNN branch cannot represent cleanly. A strong result should beat the stacked edge-gate FFN warmup-cosine result; a merely larger-model effect should be checked later with a parameter-matched control.

## Key Characteristics

- Model variant: `parallel-cnn-attn-fuse`.
- Shared encoder: `GhostShuffleBlock x3`.
- Local branch: `GhostShuffleBlock x3`.
- Global branch: `BoardAttentionStack x3`, each with 4 heads, 16 head dim, and FFN hidden size 64.
- Fusion: concatenate local and global branch outputs, then `Conv2d 1x1 512 -> 256`, BatchNorm, Hardswish.
- Output contract: `value_logit = material_prior_logit + positional_residual_logit`.
- Material prior: raw board material diff multiplied by `log(10) / 4`.
- Residual head receives fused activations only and does not receive material diff or material feature.
- Scheduler recipe: warmup-cosine, 5 warmup epochs, no warm restart.
- Design doc: `docs/designs/2026-05-26.02-parallel-cnn-attention-fusion.md`.

## Verdict Rationale

The primary objective failed: the model did not beat the stacked edge-gate FFN warmup-cosine reference. It is larger (`589205` parameters versus `457685`), slower, worse on full test BCE by `0.0029836720391506`, and worse on ECE by about `0.0020659768095592`.

The result is still informative. It lands in the weak-gain BCE band relative to the one-layer edge-gate and no-attention controls, and the best sampled validation point occurs at epoch 100, so the architecture continued to learn late. However, the cost-adjusted outcome is not favorable, and calibration is meaningfully worse than both attention references.

The unusually heavy gradient-norm tail points to an optimization-conditioning issue. Candidate causes include the fixed material-prior plus residual objective, branch fusion, the parallel branch topology, FFN attention depth, larger parameter count, or a mismatch between empirical labels and the assumed material-score prior.

## Issues And Anomalies

- The first smoke run emitted a `torch.compile` cudagraph partition warning from a `DeviceCopy` in `_material_diff_from_board`. Root cause: the new top-level material weights were initially passed as a CPU tensor during forward. Resolution: register `ValueOnlyModel.material_weights` as `persistent=False`, so it moves with the model without adding a checkpoint key.
- Fresh-context review found that `cpe train --resume` constructed the model from the CLI default variant before reading checkpoint metadata. Resolution: resolve `model_variant` from the checkpoint before constructing `ValueOnlyModel` when `--resume` is used.
- During the full run, W&B showed substantially larger `train/grad_norm` spikes than prior warmup-cosine controls. This may indicate poorer optimization conditioning from the parallel branch, the fixed material-prior plus residual objective, branch fusion, larger parameter count, or a combination of these changes. The gradient clipping protects updates but does not remove the underlying signal.
- Full evaluation showed worse calibration than the stacked edge-gate FFN and one-layer edge-gate references despite acceptable BCE improvement over weaker controls. This makes the fixed material-prior assumption worth auditing against the dataset labels.

## Follow-Up

- Analyze dataset label calibration against material scores in `docs/todos/2026-05-26.00.analyze-dataset-label-calibration-against-material-scores.md`.
- Inspect calibration and the fixed material-prior scale before running another material-prior residual variant.
- If gains are weak, inspect branch activations and compare local/global feature redundancy with `cpe analyze-rank`.
- Compare gradient-norm tails in follow-up ablations to separate the effects of material prior, branch fusion, FFN attention depth, and parameter count.
