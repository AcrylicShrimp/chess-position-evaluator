## Goal

Run a 100-epoch promotion experiment for the depth-preserving refresh architecture at `lr=0.00075`.

This experiment tests whether the successful `lr=0.00075` 40-epoch gate translates into a better full-run result than the previous `lr=0.0005` 100-epoch run. The decision this informs is whether the depth-refresh architecture should be rejected as weak, or whether its previous full-run result was materially limited by an overly conservative learning rate.

## Hypothesis Or Expectation

The `lr=0.00075` run should remain stable through 100 epochs and improve full validation/test BCE over the `lr=0.0005` 100-epoch reference.

Expected positive evidence:

- no epoch-20s divergence comparable to the `lr=0.001` gate failure
- sampled validation better than the `lr=0.0005` trajectory by epoch 40 and remaining competitive through epoch 100
- full validation BCE better than `0.6031740756086265`
- full test BCE better than `0.6032937549366552`
- ideally closing some of the gap to the previous six-attention funnel test BCE `0.602155957334376`

## Key Characteristics

- Model variant: `funnel-cnn224-160-128-attn6-refresh3-edgegate`.
- Architecture is unchanged from the depth-refresh learning-rate ablations.
- Training recipe is unchanged from the prior full run except `lr=0.00075`.
- `eta_min` is intentionally kept at `0.000001` to isolate the base-LR change.
- This is a promotion run from the successful `lr=0.00075` 40-epoch gate.

## Verdict Rationale

Pass. The promotion run met the primary success criteria and materially changes the assessment of this architecture.

The `lr=0.00075` run was stable through 100 epochs, beat the `lr=0.0005` 100-epoch reference by `0.002516` full validation BCE and `0.002527` full test BCE, and also beat the previous six-attention funnel. It slightly improved on the cleaned stacked reference on both validation and test BCE, with lower ECE.

The correct conclusion is not that depth-refresh is weak. The earlier weak result was strongly learning-rate limited.

## Issues And Anomalies

- The 100-epoch trajectory was slower than the 40-epoch `lr=0.00075` gate through the first half of training, but it recovered strongly after epoch 60.
- Final attention-stage rank still concentrates somewhat (`depth_refresh_2 eff_rank=121/128`), so the architecture is not fully representation-balanced.

## Follow-Up

- Treat `lr=0.00075` as the current default candidate for this depth-refresh family.
- Reconsider this architecture as a viable baseline, not a failed branch.
- Next useful experiment: either replicate `lr=0.00075` once to estimate seed variance, or apply the same LR to the strongest stacked edge-gate FFN family for a fair optimizer-level comparison.
