## Goal

Test whether the latest stacked attention result can be explained by the scheduler change alone.

This experiment reruns the 256-channel non-attention baseline with the same warmup-cosine no-restart scheduler that produced the strong stacked edge-gated attention result.

## Hypothesis Or Expectation

If the scheduler change alone explains most of the improvement, then the no-attention baseline should approach the stacked attention full test BCE `0.6010760372153482`.

If the no-attention model remains much worse, then the stacked result requires either added capacity, attention-specific mixing, or both.

## Key Characteristics

- Architecture variant: `ValueOnlyModel` with `CHANNELS=256` and `model_variant=no-attention`.
- Board mixer: `IdentityBoardAttention`, so the trunk has no full-board self-attention parameters.
- Expected `ValueOnlyModel` parameters: `155,861`.
- Scheduler variable: use `warmup-cosine`, matching the successful stacked run.
- Warmup behavior: linearly ramp learning rate from `0.1 * lr` to `lr` across the first `5` epochs.
- Decay behavior: after warmup, cosine decay monotonically approaches `eta_min=1e-6` with no restart.
- Peak learning rate: `0.0005`.
- Dataset and split policy: unchanged.

## Verdict Rationale

Verdict is `Informational` because this is a controlled ablation without a product pass/fail gate.

The result lands in the pre-registered "scheduler contributes but is not sufficient" band: full test BCE improved over the original no-attention baseline by `0.0012359327121935`, which is larger than the `0.001` improvement threshold.

It also lands in the pre-registered "attention/capacity still required" band: full test BCE remained `0.0078066860724583` worse than the stacked edge-gate FFN warmup-cosine run, exceeding the `0.005` gap threshold.

This separates the effects cleanly enough for the current research loop: warmup-cosine is a real training-regime improvement, but the stacked attention result cannot be attributed to the scheduler alone.

## Issues And Anomalies

- Validation full eval emitted `validation_split_is_model_selection_data`, which is expected because the validation split selected the best checkpoint.
- Full validation/test BCE were higher than the sampled W&B validation loss, so final comparisons should use the full eval JSON reports rather than the sampled training metric.

## Follow-Up

- Keep warmup-cosine as the preferred scheduler unless later controlled ablations contradict it.
- Run a capacity-matched no-attention ablation if we need to separate attention-specific mixing from parameter count.
- Run a one-layer edge-gate or no-static 4-head attention warmup-cosine ablation if we need a cleaner scheduler-only comparison inside weak-attention variants.
- Continue stacked attention research; this ablation supports attention/capacity as a real contributor.
