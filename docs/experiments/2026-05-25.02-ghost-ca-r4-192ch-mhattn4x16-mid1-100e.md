## Goal

Test whether board self-attention remains useful when the trunk width is reduced enough to make total parameter count comparable to the pre-attention baseline.

This experiment is meant to inform whether the previous attention gain was mostly a capacity increase or whether the global square-to-square mixer is structurally useful.

## Hypothesis Or Expectation

The 192-channel multi-head attention model should remain competitive with the 256-channel non-attention baseline. Beating the prior 256-channel single-head attention run would be a stronger positive signal, but is not required for the architecture direction to remain promising.

## Key Characteristics

- Architecture variant: `ValueOnlyModel` with `CHANNELS=192`.
- Trunk: six `GhostShuffleBlock` modules with coordinate attention.
- Board attention: one `NaiveBoardSelfAttention` module after the third trunk block.
- Attention layout: `4` heads, `16` dimensions per head, total projection dimension `64`.
- Attention scaling: per-head scale `16 ** -0.5`.
- Total parameters: `158,965`.
- Attention parameters: `49,152`.
- Comparison targets:
  - 256-channel non-attention baseline: `155,861` parameters.
  - 256-channel single-head attention run: `221,397` parameters.

## Verdict Rationale

The verdict is `Fail` for the primary comparison target: the parameter-matched 192-channel, 4-head attention model did not remain competitive with the 256-channel non-attention baseline on full validation or full test evaluation.

The run completed successfully, but the full test BCE is `0.614751371`, which is worse than the baseline test BCE `0.610118656` by `0.004632714`. Test CP MAE is also worse by about `11.99cp`.

This does not prove that multi-head attention is bad. The experiment changes trunk width and attention layout at the same time, so the negative result could come from the 192-channel reduction, the multi-head split, or their interaction.

## Issues And Anomalies

- Full-split evaluation was materially worse than sampled validation suggested.
- The combined change does not isolate whether width reduction or multi-head layout caused the regression.

## Follow-Up

- Do not adopt `192ch + 4x16 attention` as the next default.
- Consider a 192-channel non-attention control if parameter-matched attribution matters.
- Consider a 256-channel multi-head attention run if the next question is single-head versus multi-head at fixed trunk width.
- Consider a less severe width reduction, such as 224 channels, if exact parameter matching is less important than keeping enough trunk capacity.
