## Goal

Train a lighter parallel CNN/attention model that keeps the useful no-material direction while testing two structural changes:

- Replace q+k dynamic edge gating with k-only edge gating.
- Replace early 256-channel fusion with late low-dimensional local/global evidence fusion.

This experiment is meant to decide whether the parallel branch design was slow and underperforming because the branch fusion was too wide/early and because q+k edge gates were doing unnecessary work.

## Hypothesis Or Expectation

The k-only gate should preserve most of the useful content-conditioned relation signal while reducing gate parameters and computation.

Late evidence fusion should force the CNN and attention branches to each produce compact value evidence before the MLP combines them. If the previous fusion layer was too unconstrained or poorly aligned, this should improve or at least match the previous no-material parallel fusion run while being smaller and faster.

## Key Characteristics

- Model variant: `parallel-cnn-attn-kedge-lateevidence-no-material`.
- Shared encoder: `GhostShuffleBlock x3`.
- Local branch: `GhostShuffleBlock x3`.
- Global branch: `BoardAttentionStack x3`, each with 4 heads, 16 head dim, FFN hidden size 64, and k-only dynamic edge gates.
- Fusion: no wide `Conv2d 1x1 512 -> 256` fusion layer.
- Local evidence projection: `Conv2d 1x1 256 -> 2`, BatchNorm, Hardswish.
- Global evidence projection: `Conv2d 1x1 256 -> 2`, BatchNorm, Hardswish.
- Value head: flatten 4 evidence planes and run `Linear(256 -> 64 -> 1)`.
- Explicit material-score input: removed.
- Additive material prior: not used.
- Dataset: strict-filtered processed data under `data/processed`.
- Scheduler recipe: warmup-cosine, 5 warmup epochs, no warm restart.

## Verdict Rationale

Fail. The pre-registered fail condition was a full test BCE regression greater than about `0.0004` versus the previous no-material parallel fusion run. This run regressed by about `0.00263` full test BCE and also worsened full test ECE by about `0.00069`.

The implementation did achieve its efficiency goal: parameter count dropped from `589205` to `463065`, and eager forward time at batch 2048 improved from `14.8307 ms` to `12.9394 ms`. However, the accuracy and calibration losses are too large for this to replace the previous no-material parallel fusion architecture.

The most likely interpretation is that the 2+2 late-evidence planes are too narrow, q-side edge gate information matters, or both. This experiment does not identify which axis caused the loss because it changed k-only gating and late evidence fusion together.

## Issues And Anomalies

- Source code was uncommitted at the moment full training started. Record the eventual commit hash after committing the implementation and log.
- No training instability observed during smoke or full training.
- The experiment intentionally changed two axes at once: q+k to k-only edge gates and wide fusion to narrow late evidence. Follow-up ablations are required to separate those effects.

## Follow-Up

- Test late evidence with wider evidence projections, for example 4+4 or 8+8 planes, while keeping q+k edge gates.
- Test k-only gates while keeping the previous wide no-material fusion path to isolate whether q-side gate removal is harmful.
- Keep this variant as a speed/parameter reference, not as an accuracy candidate.
- Commit the implementation and experiment log as a focused work unit after final local verification.
