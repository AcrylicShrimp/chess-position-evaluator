# Funnel Interleaved Attention

## Design Goal

Test whether interleaving local CNN refresh blocks between narrow edge-gated attention blocks prevents the late-stage spectral concentration observed in the `funnel-cnn224-160-128-attn6-edgegate` experiment.

The specific experimental question is:

- Can `C128` attention remain fast and useful if three attention blocks are separated by `C128` GhostShuffle refresh blocks instead of being stacked six times consecutively?

## Agreed Model

Add one value-only model variant:

- `funnel-cnn224-160-128-interleave-attn3-edgegate`

The model keeps the same wide-to-narrow CNN encoder and material-difference scalar value-head semantics as the previous funnel experiment. The only architectural change after the narrow CNN stage is replacing `Attention C128 x6` with `[Attention C128 -> GhostShuffleBlock C128] x3`.

Data flow:

[B, 160, 8, 8]

[B, 128, 8, 8]

## Key Invariants And Boundaries

- Keep board spatial size fixed at `8x8`.
- Keep the same `224 -> 160 -> 128` compression path as the previous funnel experiment.
- Keep q+k content-aware edge gates.
- Keep attention heads, head dimension, FFN hidden size, scheduler recipe, and material-difference semantics unchanged.
- Do not interleave attention before the `C128` compression point in this slice.
- Treat this as a value-only architecture. Do not change `ModelFull` or policy-head behavior.

## Tradeoffs

Benefits:

- Directly tests whether the previous funnel failure came from consecutive narrow attention blocks rather than the `C128` bottleneck itself.
- Restores local/spatial refresh between global mixing operations.
- Reduces parameter count compared with both the stacked baseline and the six-attention funnel model.
- Should preserve most of the narrow-attention speed advantage.

Risks:

- Reducing attention depth from six to three may undercut global reasoning capacity.
- `C128` may still be too narrow even with CNN refresh.
- The added CNN refresh blocks may improve rank shape without improving BCE.
- A much smaller parameter count may make the comparison less controlled than the previous funnel run.

## Success Criteria

Primary references:

- cleaned-data `stacked-edge-gate-ffn`
  - full validation BCE: `0.6008478613148978`
  - full test BCE: `0.6009210722625944`
  - parameter count: `457685`
- previous `funnel-cnn224-160-128-attn6-edgegate`
  - full validation BCE: `0.6020264165573299`
  - full test BCE: `0.602155957334376`
  - parameter count: `474197`

Interpretation:

- Strong pass: full test BCE improves over `0.6009210722625944`.
- Architecture pass: full test BCE improves meaningfully over `0.602155957334376` and post-training rank analysis no longer shows a severe final-block spectral concentration.
- Fail: full test BCE remains near or worse than the previous funnel result.
