# Stacked Edge-Gated Attention With FFN

## Design Goal

Test whether the edge-gated board attention mechanism underperformed because it was too shallow.

The prior one-layer edge-gate run learned non-trivial dynamic relation gates and beat no-static 4-head attention on full BCE, but missed the static relation-bias target and worsened calibration. This design increases edge-gated attention depth while preserving the core model width, head layout, insertion point, data split, and training recipe.

## Agreed Architecture

Replace the single board attention module with a three-layer stack at the existing insertion point after trunk block 3.

Each stack layer is:

```text
x = NaiveBoardSelfAttention(x)
x = BoardAttentionFFN(x)
```

The FFN is a per-square channel MLP implemented with `1x1` convolutions:

```text
BoardAttentionFFN:
  residual + Conv2d(256 -> 64, 1x1, bias=False)
             BatchNorm2d(64)
             Hardswish
             Conv2d(64 -> 256, 1x1, bias=False)
```

This FFN does not mix board tokens directly. Token mixing remains the job of dense `64 x 64` board attention. The FFN adds per-square channel transformation after each global attention pass.

## Parameter Contract

Constants:

```text
ATTENTION_LAYERS = 3
ATTENTION_FFN_HIDDEN = 64
```

Expected `ValueOnlyModel` parameter count:

```text
current one-layer edge-gate model: 223,573
one edge-gated attention layer:     67,712
one hidden-64 FFN:                  32,896
base without attention:            155,861
new total:                         457,685
```

This is not a parameter-matched experiment. It is a depth/capacity experiment.

## Boundaries And Invariants

- Keep `CHANNELS = 256`.
- Keep `ATTENTION_HEADS = 4`.
- Keep `ATTENTION_HEAD_DIM = 16`.
- Keep dense `64 x 64` board attention.
- Keep the existing edge-gate relation and distance contract.
- Keep the attention insertion point after trunk block 3.
- Apply the same hard cutover to both `ValueOnlyModel` and `ModelFull`.
- Do not preserve checkpoint compatibility with the one-layer edge-gate model.
- Do not add static relation bias back into this experiment.
- Do not change dataset splits or the best-recipe training hyperparameters.

## Verdict Criteria

Primary comparison target:

```text
ghost-ca-r4-256ch-mhattn4x16-edgegate-rel9dist8-mid1-bestrecipe-100e
```

Primary outcome precedence:

1. Fail if full test BCE does not beat `0.6077498676321618`.
2. Inconclusive if full test BCE beats `0.6077498676321618` but full validation BCE does not beat `0.6076844683747489`.
3. Pass if full validation BCE beats `0.6076844683747489` and full test BCE beats `0.6077498676321618`.

Strong practical pass additionally requires beating the static relation-bias run:

```text
validation BCE < 0.6075506562081671
test BCE       < 0.6075916815231461
```

Calibration warning:

```text
full test ECE > 0.0017807864956025888
```

## Smoke Gate

Before a full run, run the target-batch smoke job:

```text
epochs = 1
steps = 128
batch = 2048
train_workers = 4
val_workers = 2
```

Stop before a full run if:

- the smoke job OOMs
- the post-warmup tqdm rate after step 32 is below `8` steps/s
- the model output or loss becomes non-finite

Do not silently change the batch size or training recipe to force the experiment through.

## Test Plan

- `BoardAttentionStack` has exactly three layers.
- Each stack layer contains `NaiveBoardSelfAttention` and `BoardAttentionFFN`.
- `ValueOnlyModel` and `ModelFull` both use `BoardAttentionStack`.
- Trunk call order remains block 0, block 1, block 2, board attention, block 3, block 4, block 5.
- Zeroing all stack branch parameters makes the stack an identity function.
- `ValueOnlyModel` parameter count is exactly `457,685`.
- Existing edge-gate tests continue to cover geometry, zero initialization, composition, and zero-gate equivalence.

## Tradeoffs And Rejected Alternatives

- Wider FFNs were rejected for this slice because they quickly turn the experiment into a large parameter-count change.
- Static relation bias was rejected because the question is specifically whether dynamic edge-gated attention improves with more depth.
- Moving the attention insertion point was rejected because it would confound depth with trunk placement.
- Parameter matching was rejected because the user-approved question is capacity/depth, not equal-size architecture comparison.
