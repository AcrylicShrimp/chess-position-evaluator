# Design: Funnel Depth-Preserving Attention Refresh

## Goal

Test whether CNN refresh blocks can reduce late-stage attention spectral concentration without reducing narrow attention depth.

The previous `funnel-cnn224-160-128-attn6-edgegate` run preserved six `C128` attention blocks but showed a strongly top-heavy final attention spectrum and worse BCE than the cleaned stacked baseline. The later `funnel-cnn224-160-128-interleave-attn3-edgegate` run improved final-stage rank shape and ECE, but reduced attention depth from six blocks to three and regressed further on BCE.

This design preserves the six attention blocks and adds refresh blocks after every two attention blocks.

## Agreed Architecture

Model variant:

- `funnel-cnn224-160-128-attn6-refresh3-edgegate`

Trunk:

```text
input board + coords
24 -> 224 initial conv
Ghost C224 x2
224 -> 160 projection
Ghost C160 x2
160 -> 128 projection
Ghost C128 x2
[Attention C128 -> Attention C128 -> Ghost refresh C128] x3
ValueHead C128
```

Attention settings:

- heads: `4`
- head dim: `16`
- FFN hidden: `64`
- edge gate mode: `qk`
- total attention blocks: `6`
- refresh blocks: `3`

## Invariants And Boundaries

- Keep dataset, split, optimizer, scheduler, batch size, and material-difference value-head semantics matched to the previous funnel and cleaned stacked experiments.
- Do not change input channels, label semantics, material-difference calculation, or value-head contract.
- Treat ECE as secondary; BCE and CP-equivalent MAE remain the main model-quality metrics.
- Treat rank analysis as diagnostic evidence, not as a replacement for validation/test BCE.
- Run this as a gated experiment before a full 100-epoch commitment.

## Tradeoffs

- Preserving six attention blocks isolates the value of refresh blocks better than the three-attention interleaved run.
- Adding three refresh blocks increases parameters above the previous six-attention funnel, so a win must justify both the extra capacity and compute.
- The experiment still does not isolate `C128` width from the funnel encoder; it only tests whether the final-stage concentration can be repaired without reducing attention depth.

## Rejected Alternatives

- Continue the three-attention interleaved family: rejected for now because it clearly trails both references on BCE.
- Make calibration the primary objective: rejected because ECE improved while BCE and CP-equivalent MAE regressed.
- Return immediately to parallel fusion: deferred because the no-material parallel fusion family improved internally but still trails the cleaned stacked baseline on BCE.

## Gate Criteria

Run a 40-epoch gate before extending to 100 epochs.

Extend only if:

- sampled validation at epoch 40 is better than the previous six-attention funnel epoch-40 value `0.6072`
- preferred margin: `<= 0.6070`
- stronger signal: `<= 0.6067`
- rank analysis on the best gate checkpoint does not repeat the previous final attention `rank>10% = 21/128` concentration
- value-head evidence balance does not regress materially from the previous funnel

Fail or stop early if:

- sampled validation is clearly worse than the previous funnel trajectory through epoch 40
- rank shape remains concentrated despite refresh
- training instability, non-finite outputs, or validation blow-up appears
