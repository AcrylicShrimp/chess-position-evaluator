# Attention Relation Bias

## Design Goal

Add a minimal chess-geometry prior to the current best board self-attention model by adding learnable relation and distance biases directly to attention logits.

The experiment should answer whether the 256-channel, 4-head attention model benefits from explicit square-to-square geometry while preserving the current best training recipe and nearly the same parameter count.

## Current Model

The current best model uses:

- `CHANNELS = 256`
- `BLOCKS = 6`
- `ATTENTION_AFTER_BLOCK = 3`
- `ATTENTION_HEADS = 4`
- `ATTENTION_HEAD_DIM = 16`
- `ATTENTION_DIM = 64`

The attention module computes full-board scaled dot-product attention over 64 board squares:

```text
logits[b, h, i, j] = q[b, h, i] @ k[b, h, j] / sqrt(16)
weights = softmax(logits, dim=-1)
context = weights @ v
output = x + out_proj(context)
```

This gives direct square-to-square interaction, but the logits do not know whether two squares share a rank, file, diagonal, pawn-attack geometry, or distance bucket except through learned trunk features.

## Agreed Architecture

Add two zero-initialized learnable bias tables:

```text
rel_bias:  [heads, num_relations] = [4, 9]
dist_bias: [heads, 8]
```

The new logit equation is:

```text
logits[b, h, i, j]
  = q[b, h, i] @ k[b, h, j] / sqrt(16)
  + relation_bias_sum[h, i, j]
  + distance_bias[h, distance[i, j]]
```

Where:

```text
relation_bias_sum[h, i, j]
  = sum(relation_mask[r, i, j] * rel_bias[h, r] for r in relations)
```

The relation masks and distance index are fixed board-geometry buffers, not parameters.

## Relation Contract

Use side-to-move-relative board coordinates. The Python encoder vertically flips black-to-move positions, so row `0` is the side-to-move home rank direction and row `7` is forward from that side's perspective.

For attention pair `(i, j)`:

- `i` is the query square.
- `j` is the key/value square being attended to.
- `dr = row(j) - row(i)`.
- `df = file(j) - file(i)`.

Relations are multi-label and additive. A pair may receive several relation biases.

Use these relation names and definitions:

| Relation | Definition |
| --- | --- |
| `same_square` | `dr == 0 and df == 0` |
| `same_rank` | `dr == 0 and df != 0` |
| `same_file` | `df == 0 and dr != 0` |
| `same_diagonal` | `abs(dr) == abs(df) and dr != 0 and df != 0 and dr == df` |
| `same_anti_diagonal` | `abs(dr) == abs(df) and dr != 0 and df != 0 and dr == -df` |
| `knight_move` | `{abs(dr), abs(df)} == {1, 2}` |
| `king_adjacent` | `max(abs(dr), abs(df)) == 1` |
| `our_pawn_attack_geometry` | `dr == 1 and abs(df) == 1` |
| `their_pawn_attack_geometry` | `dr == -1 and abs(df) == 1` |

Do not call the pawn relations `white_*` or `black_*` in implementation. The model input is side-to-move relative, so `our_*` and `their_*` are the stable names.

`same_square` is excluded from line relations by the `!= 0` clauses above. This prevents a self-square pair from accumulating rank, file, diagonal, and anti-diagonal biases at once.

`king_adjacent` overlaps with rank, file, diagonal, anti-diagonal, and pawn geometry when applicable. This is intentional.

## Distance Contract

Use Chebyshev distance:

```text
distance[i, j] = max(abs(dr), abs(df))  # 0..7
```

`same_square` also receives `distance = 0`. This overlap is intentional because self-square identity and distance bucket are separate learned signals.

## Implementation Shape

Keep the implementation inside `NaiveBoardSelfAttention`.

Recommended buffers:

```text
relation_masks_flat: [9, 4096]
distance_index_flat: [4096]
```

Use this fixed flattening order:

```text
token_index = row * 8 + file
pair_index = query_token_index * 64 + key_token_index
```

Recommended forward calculation:

```text
base_logits = matmul(q, k) * scale

relation_bias = rel_bias @ relation_masks_flat
relation_bias = relation_bias.reshape(heads, 64, 64)

distance_bias = dist_bias.index_select(dim=1, index=distance_index_flat)
distance_bias = distance_bias.reshape(heads, 64, 64)

logits = base_logits + relation_bias[None] + distance_bias[None]
weights = softmax(logits, dim=-1)
```

Convert relation masks, relation bias output, and distance bias output to the logits dtype inside `forward` so mixed precision or dtype changes do not unexpectedly promote the attention logits.

The attention module is board-specific. It should explicitly reject non-`8x8` spatial inputs rather than silently applying invalid geometry buffers.

## Invariants And Boundaries

- Keep trunk width at `256`.
- Keep `4` heads and `16` dimensions per head.
- Keep the attention insertion point after block index `2`.
- Keep the model input and output contracts unchanged.
- Initialize `rel_bias` and `dist_bias` to exact zeros.
- With both bias tables at zero, the new module must be functionally equivalent to the current 4-head attention module.
- Do not add LayerNorm, dropout, FFN, local windows, piece-conditioned masks, or graph-attention message schemas in this slice.
- Checkpoint compatibility with older attention checkpoints is intentionally not preserved.

## Test Plan

Add focused unit tests for:

- `rel_bias.shape == (4, 9)` and exact zero initialization.
- `dist_bias.shape == (4, 8)` and exact zero initialization.
- Relation mask correctness for representative square pairs:
  - same square
  - same rank
  - same file
  - same diagonal
  - same anti-diagonal
  - knight move
  - king adjacent
  - our pawn attack geometry
  - their pawn attack geometry
- Distance index correctness for representative square pairs.
- Nonzero additive bias composition for a pair that matches more than one relation plus a distance bucket.
- Zero-bias equivalence against the previous logit calculation while Q/K/V/out projection weights are nonzero.
- Existing attention shape, finiteness, residual identity when all branch parameters are zeroed, and model placement tests.
- Explicit rejection of non-`8x8` attention inputs.

## Tradeoffs And Rejected Alternatives

- A single exclusive `relation_type(i, j)` was rejected because chess square geometry is naturally multi-label.
- Color-named pawn relations were rejected because model inputs are side-to-move relative.
- Dynamic geometry generation in `forward` was rejected because the model is board-specific and fixed buffers are simpler, deterministic, and easier to test.
- Piece-conditioned relation bias was rejected for this slice because it would mix board occupancy features into the attention prior and make the experiment harder to interpret.
- Graph attention was rejected for this slice because relation bias is a cheaper, more interpretable intermediate step.
- A single unseeded run may be noisy for tiny metric deltas; if both full validation and test improve only marginally, prefer replication before making a broader architecture decision.
