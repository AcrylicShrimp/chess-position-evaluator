# Parallel CNN Attention Fusion

## Design Goal

Define a larger value-model architecture experiment that separates local CNN processing from global attention processing, then fuses both branches before the value head.

The experiment should answer whether a parallel local/global trunk is stronger than inserting an attention stack into a single CNN trunk.

## Agreed Architecture

Add a new model variant named:

```text
parallel-cnn-attn-fuse
```

The value-only model should use this high-level flow:

```text
Input board [B, 20, 8, 8]
  -> material_diff from raw board
  -> material_prior_logit

Input board [B, 20, 8, 8]
  -> AddCoords
     [B, 24, 8, 8]
  -> Stem Conv 3x3, BatchNorm, Hardswish
     [B, 256, 8, 8]
  -> Shared CNN encoder: GhostShuffleBlock x3
     [B, 256, 8, 8]
  -> split

     Local branch:
       GhostShuffleBlock x3
       [B, 256, 8, 8]

     Global branch:
       BoardAttentionStack x3
         each layer:
           NaiveBoardSelfAttention(heads=4, head_dim=16)
           BoardAttentionFFN(hidden=64)
       [B, 256, 8, 8]

  -> Fusion:
       concat(local, global)
       [B, 512, 8, 8]
       Conv1x1 512 -> 256, BatchNorm, Hardswish
       [B, 256, 8, 8]

  -> residual value head
     positional_residual_logit [B, 1]

Output:
  value_logit = material_prior_logit + positional_residual_logit
```

## Material Prior Contract

The model output must be an additive logit composition:

```text
final_value_logit = material_prior_logit + positional_residual_logit
```

The material prior should be computed from the raw board piece channels, before coordinate channels or trunk activations are added.

Use fixed centipawn-compatible scaling:

```text
material_cp = material_diff * 100.0
material_prior_logit = log(10) * material_cp / 400.0
material_prior_logit = material_diff * (log(10) / 4.0)
```

The residual head should not receive the material feature again. This keeps the contract clear:

- material branch: material-only prior
- network branch: positional, tactical, structural, and long-range residual
- final output: additive composition in logit space

## Key Invariants And Boundaries

- Input and output shape remain unchanged for training and evaluation.
- The checkpoint must record `model_variant=parallel-cnn-attn-fuse`.
- The value head output remains a logit for `binary_cross_entropy_with_logits`.
- Material is added in logit space, not probability space.
- The residual head must not receive an explicit material feature or material diff.
- Material prior calculation must not use coordinate-augmented tensors or trunk activations.
- The global branch receives shared shallow CNN features, not raw board tokens.
- The local and global branches are independent after the shared three-block encoder.
- Fusion is late and explicit: concatenate branch outputs, then project back to `256` channels with a `1x1` convolution.
- Policy-head behavior is out of scope for this experiment.

## Tradeoffs

- The shared shallow CNN encoder gives attention richer square tokens than raw board input while keeping the global branch from being dominated by a deep CNN trunk.
- Late fusion preserves separate local and global feature streams until the final trunk output.
- Keeping FFN inside the attention branch matches the current stacked attention block and isolates the parallel-branch architecture change more cleanly.
- The fusion projection adds about `131,072` parameters, so a strong result may require a later parameter-matched control.
- Fixed material scaling preserves interpretability, but a future learnable-material-scale ablation may fit the dataset slightly better.

## Rejected Alternatives

- Raw-board attention branch: rejected for the first experiment because query/key/value projections would need to learn local square context from very low-level inputs.
- Deep shared CNN before attention: rejected because the global branch would inherit too much local-CNN bias and have less room to specialize.
- Removing FFN from the attention branch in this first variant: deferred because it would mix branch-architecture effects with attention-block contract changes.
- Concatenating material into the residual head: rejected because it weakens the residual-over-material interpretation.
- Adding material in probability space: rejected because training and evaluation use logits with `binary_cross_entropy_with_logits`.

## Failure Modes To Watch

- Fusion capacity may explain gains rather than the local/global split.
- The attention branch may learn redundant features if the shared encoder is too strong.
- The fixed material prior may be miscalibrated against the dataset and force the residual head to learn a global scale correction.
- Branch outputs may have mismatched activation statistics; BatchNorm in the fusion projection should be retained for the first experiment.
- If performance improves but ECE regresses, the material prior plus residual composition may need calibration analysis.

## Experiment Criteria

Use the same training recipe as the current best attention-family experiments unless intentionally changed:

```text
batch = 2048
lr = 0.0005
wd = 0.0001
scheduler = warmup-cosine
warmup_epochs = 5
warmup_start_factor = 0.1
eta_min = 1e-6
grad_clip = 5
epochs = 100
steps = 1024
```

Suggested interpretation bands for full test BCE:

- Strong win: `< 0.6010`, beating the stacked edge-gate FFN warmup-cosine result.
- Promising: `0.6010..0.6030`, meaning the parallel local/global split is a strong signal.
- Weak gain: `0.6030..0.6061`, better than the one-layer warmup-cosine ablation but not clearly better than stacked attention.
- Fail: `>= 0.6061`, similar to or worse than the one-layer warmup-cosine ablation.
