# Principled Parallel Branch Fusion

## Design Goal

Define the first controlled replacement for naive parallel local/global fusion.

The target is the `parallel-cnn-attn-fuse` architecture family. The immediate experiment tests whether explicitly aligning local CNN and global attention branch representations before fusion improves stability and quality.

## Current Model

The current parallel trunk is:

```text
x0 = stem(add_coords(board))
s  = shared_cnn_3blocks(x0)

l  = local_cnn_3blocks(s)
g  = attention_stack_3blocks(s)

z  = HSwish(BN(Wf [l; g]))
y  = value_head(z, material_features)
```

The fusion layer can be decomposed as:

```text
z_ij = HSwish(BN(W_l l_ij + W_g g_ij))
```

for each board square `(i, j)`.

This assumes the branch tensors are directly comparable channel vectors at each square. The diagnostic evidence does not support that assumption:

- local/global flattened cosine mean: `0.018594982776641844`
- local RMS: `0.6422484329459918`
- global RMS: `0.9440983744942347`
- the parallel model is worse than the clean stacked edge-gate FFN baseline by `0.001788217617571354` BCE on the diagnostic validation prefix

## Branch Mismatch Analysis

The local branch applies three `GhostShuffleBlock` modules after the shared encoder:

```text
l = C_local(s)
```

This path is local and convolutional. It repeatedly mixes nearby spatial evidence, coordinate attention, channel shuffle, and partial-channel transformations. Its channels are expected to encode local motifs and local board geometry.

The global branch applies three attention blocks:

```text
g_0 = s
g_{t+1} = FFN_t(Attention_t(g_t))
g = g_3
```

Each attention block is residual:

```text
Attention(x) = x + O(softmax(Qx Kx / sqrt(d) + edge_gate(q, k)) Vx)
FFN(x)       = x + Conv1x1(HSwish(BN(Conv1x1(x))))
```

This path keeps the shared tensor as a residual anchor while injecting global square-to-square interactions. Its channels are expected to encode remote interactions and attention-routed context.

Therefore `l_ij` and `g_ij` have the same shape but not necessarily the same basis, scale, or semantics. Direct concatenation plus `Conv1x1` makes one static per-square mixer responsible for three jobs at once:

1. normalize branch scales
2. map branch-specific bases into a shared basis
3. decide how much local/global information to keep

The first replacement should separate those jobs.

## Agreed First Test

Add a new model variant:

```text
parallel-cnn-attn-aligned-add
```

Use the same shared, local, and global branches as `parallel-cnn-attn-fuse`, but replace concat fusion with aligned add fusion:

```text
s = shared_cnn_3blocks(x0)
l = local_cnn_3blocks(s)
g = attention_stack_3blocks(s)

l_hat = P_l(N_l(l))
g_hat = P_g(N_g(g))

z = HSwish(N_z(l_hat + g_hat))
y = value_head(z, material_features)
```

Where:

```text
N_l = BatchNorm2d(256)
N_g = BatchNorm2d(256)
P_l = Conv2d(256, 256, kernel_size=1, bias=False)
P_g = Conv2d(256, 256, kernel_size=1, bias=False)
N_z = BatchNorm2d(256)
```

Implementation shape:

```text
local_aligner  = BatchNorm2d(256), Conv1x1(256 -> 256, bias=False)
global_aligner = BatchNorm2d(256), Conv1x1(256 -> 256, bias=False)
fuse           = BatchNorm2d(256), Hardswish
```

## Key Invariants And Boundaries

- `l`, `g`, `l_hat`, `g_hat`, and `z` are all shaped `[B, 256, 8, 8]`.
- The local and global branch modules are unchanged from the existing parallel-fusion run.
- The value head is unchanged for this first test. Material-feature removal remains a separate experiment.
- Fusion is additive after per-branch normalization and projection.
- The new variant must not alter checkpoint loading or behavior for existing variants.
- The new variant must be selectable through `cpe train --model-variant`.

## Tradeoffs

- This is a clean first test of branch-space alignment because it keeps the branch bodies and value head constant.
- Parameter count remains close to the existing concat-fusion model: two `256 -> 256` projections replace one `512 -> 256` projection.
- Additive fusion forces both branches into a shared latent basis before combination, which improves interpretability of the fusion contract.
- Additive fusion is less expressive than concat fusion because it removes arbitrary post-concat mixing. That is intentional for the first controlled test.
- Dynamic gating is deferred. If aligned add improves but remains brittle, a gated residual fusion is the next higher-capacity variant.

## Rejected Alternatives For This Slice

- Keep concat and only add BatchNorm: rejected because it still leaves basis alignment implicit.
- Add content-dependent gate now: rejected because it mixes branch-space alignment with dynamic branch selection.
- Cross-condition local and global projections: rejected as too broad for the first controlled test.
- Remove material features in the same run: rejected because it would confound branch fusion and value-head input effects.

## Failure Modes

- If performance is unchanged, the previous failure may be caused by the parallel split itself, material-feature head behavior, or insufficient branch interaction rather than branch scale/basis mismatch.
- If early training is unstable, the extra BatchNorm/projection path may need identity-like initialization or a lower learning rate.
- If BCE improves but ECE worsens, the architecture may still be overconfident on mate-clamped and sparse endgame rows.
- If aligned add is slower or similar quality, the simpler stacked edge-gate FFN remains the stronger baseline.
