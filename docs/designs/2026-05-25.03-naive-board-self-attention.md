# Naive Board Self-Attention

## Design Goal

Add the smallest useful global square-to-square interaction mechanism to the current value model so the next experiment can test whether a board-wide mixer improves chess position evaluation beyond the existing CNN/Ghost trunk.

The first slice should isolate the attention signal as much as practical:

- keep the 256-channel trunk width unchanged
- keep the six existing `GhostShuffleBlock` layers
- insert one naive self-attention module in the middle of the trunk
- avoid Transformer-specific additions until a simpler attention baseline exists

## Current Model

The current `ValueOnlyModel` flow is:

```text
input [B, 20, 8, 8]
  -> AddCoords
  -> initial Conv2d/BatchNorm/Hardswish [B, 256, 8, 8]
  -> GhostShuffleBlock x6
  -> ValueHead
```

Each `GhostShuffleBlock` provides local convolutional mixing plus coordinate attention, but it does not provide direct token-to-token interaction between arbitrary board squares.

## Agreed Architecture

Add a `NaiveBoardSelfAttention` module with this external contract:

```text
input:  [B, 256, 8, 8]
output: [B, 256, 8, 8]
```

Insert it after the first three ghost blocks and before the last three ghost blocks:

```text
input
  -> AddCoords
  -> initial conv
  -> GhostShuffleBlock x3
  -> NaiveBoardSelfAttention(channels=256, attn_dim=64)
  -> GhostShuffleBlock x3
  -> ValueHead
```

The attention calculation is intentionally naive scaled dot-product attention over the 64 board squares:

```text
x: [B, 256, 8, 8]

q = Conv2d(256, 64, kernel_size=1, bias=False)(x)
k = Conv2d(256, 64, kernel_size=1, bias=False)(x)
v = Conv2d(256, 64, kernel_size=1, bias=False)(x)

q_tokens: [B, 64, 64]
k_tokens: [B, 64, 64]
v_tokens: [B, 64, 64]

weights = softmax(q_tokens @ k_tokens.T / sqrt(64), dim=-1)
context = weights @ v_tokens

projected = Conv2d(64, 256, kernel_size=1, bias=False)(context as [B, 64, 8, 8])
output = x + projected
```

## Invariants And Boundaries

- `CHANNELS` remains `256`.
- `BLOCKS` remains `6`.
- The value head interface remains unchanged.
- The model input contract remains `[B, 20, 8, 8]`.
- The model output contract remains raw side-to-move win-probability logits shaped `[B, 1]`.
- The attention module must not alter spatial size or trunk channel count.
- No causal mask, local mask, positional embedding, dropout, LayerNorm, Transformer FFN, or multi-head split is part of the first experiment.
- The existing coordinate channels are treated as sufficient positional signal for this first slice.
- Checkpoint compatibility with pre-attention `ValueOnlyModel` checkpoints is intentionally not preserved.

## Implementation Notes

- Keep `model.blocks` as the six ghost blocks so existing tooling can still reason about the core convolutional trunk.
- Add a separate `model.board_attention` module and call it during forward after block index `2`.
- Use default convolution initialization for the attention branch so the branch is active from the start of training.
- Add focused tests for:
  - attention module output shape
  - finite attention output on random input
  - residual identity when the attention branch is zeroed
  - model placement: six ghost blocks and one attention module after the third ghost block
  - `ValueOnlyModel` forward shape
  - `analyze_rank` hook coverage for the new attention activation
- Update `analyze_rank` intentionally so activation analysis records the attention module separately instead of silently treating it as a seventh ghost block.

## Tradeoffs

- `attn_dim=64` adds a meaningful but not dominant amount of capacity compared with full 256-dimensional Q/K/V attention.
- A serial middle insertion creates a clean first experiment but does not test richer local/global fusion.
- Omitting LayerNorm keeps the first experiment simple and isolates attention, but a later normalized attention variant may be more stable.
- Keeping `CHANNELS=256` preserves baseline comparability, but the attention branch still changes parameter count.

## Rejected Alternatives

- Full 256-dimensional attention: rejected for the first slice because it adds too much capacity relative to the current model and makes interpretation harder.
- Parallel local/global fusion block: rejected for the first slice because it introduces extra fusion design choices.
- Transformer block with LayerNorm and FFN: rejected for the first slice because it tests a larger architectural package rather than naive board attention.
- Reducing trunk channel count: rejected because it breaks direct comparison with the current baseline.
