# Funnel Narrow Attention

## Design Goal

Test whether a wide-to-narrow CNN funnel followed by a deeper narrow edge-gated attention stack is a better use of the current parameter budget than the existing wide stacked edge-gate baseline.

The specific experimental question is:

- Can `C128` attention with six repeated global-relation updates outperform or match `C256` attention with three updates when the CNN encoder first compresses local board features through `224 -> 160 -> 128` channels?

## Agreed Model

Add one value-only model variant:

- `funnel-cnn224-160-128-attn6-edgegate`

The model keeps the existing value-head semantics with a material-difference scalar and changes only the trunk shape.

Data flow:

```text
input board
[B, 20, 8, 8]

AddCoords
[B, 24, 8, 8]

initial local lift
Conv3x3 24 -> 224
BatchNorm
Hardswish
[B, 224, 8, 8]

wide CNN encoder
GhostShuffleBlock(224)
GhostShuffleBlock(224)
[B, 224, 8, 8]

compression 1
Conv1x1 224 -> 160
BatchNorm
Hardswish
[B, 160, 8, 8]

mid CNN encoder
GhostShuffleBlock(160)
GhostShuffleBlock(160)
[B, 160, 8, 8]

compression 2
Conv1x1 160 -> 128
BatchNorm
Hardswish
[B, 128, 8, 8]

narrow CNN encoder
GhostShuffleBlock(128)
GhostShuffleBlock(128)
[B, 128, 8, 8]

deep global reasoner
6x BoardAttentionBlock(
  channels=128,
  heads=4,
  head_dim=16,
  ffn_hidden=64,
  edge_gate_mode=qk
)
[B, 128, 8, 8]

value head
Conv1x1 128 -> 2
BatchNorm
Hardswish
Flatten
concat material_diff scalar
Linear(129 -> 64)
Hardswish
Linear(64 -> 1)
```

## Key Invariants And Boundaries

- Keep board spatial size fixed at `8x8`; this is channel compression only.
- Do not interleave CNN and attention in this first experiment.
- Do not change heads, head dimension, edge-gate relation geometry, scheduler recipe, or material-difference semantics.
- Keep q+k edge gates. The k-only wide-fusion experiment showed a small but persistent accuracy loss versus q+k.
- Keep a material-difference scalar in the value head, matching the current stacked baseline family rather than the no-material parallel family.
- Treat this as a value-only architecture. Do not change `ModelFull` or policy-head behavior in this slice.

## Tradeoffs

Benefits:

- Keeps parameter budget near the current stacked edge-gate baseline.
- Doubles attention reasoning depth from three to six blocks.
- Avoids the parallel-branch execution and fusion problems observed in recent experiments.
- Preserves a clean interpretation: wide local encoding, then narrow repeated global relation updates.

Risks:

- Compressing to `C128` before attention may discard local tactical features that the attention stack cannot recover.
- Deeper sequential attention may increase latency even with similar parameter count.
- The value head must support a non-global channel width; otherwise the variant will silently reintroduce `C256` assumptions.

## Rejected Alternatives

- `256 -> 192 -> 128` funnel: rejected for the first run because it raises the parameter budget to about `509k`, making the result harder to compare against the `457k` stacked baseline.
- Flat `C160 + attention6`: still useful later, but it does not test the wide local encoder plus narrow attention reasoner hypothesis as directly.
- CNN/attention interleaving: likely interesting, but rejected for this run because it would mix funnel compression, attention depth, and local/global refresh effects.
- Parallel CNN/attention branches: deprioritized because the recent experiments showed no enough speed benefit and weaker BCE than the stacked baseline.

## Success Criteria

Primary reference:

- cleaned-data `stacked-edge-gate-ffn`
- full validation BCE: `0.6008478613148978`
- full test BCE: `0.6009210722625944`
- parameter count: `457685`
- forward benchmark: `14.1824 ms`

Interpretation:

- Strong pass: full test BCE improves over `0.6009210722625944`.
- Practical pass: full test BCE is within about `0.0002` of the baseline and forward latency is similar or better.
- Fail: full test BCE regresses by at least about `0.0003` while latency is not meaningfully better.
