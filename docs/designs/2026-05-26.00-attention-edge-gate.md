# Attention Edge Gate Without Static Bias

## Design Goal

Replace the static relation and distance logit bias in board attention with a content-conditioned edge gate.

The experiment should answer whether chess geometry is more useful when it selects feature-dependent query/key gates instead of adding unconditional relation or distance constants to every matching square pair.

## Current Model

The current best model uses:

- `CHANNELS = 256`
- `BLOCKS = 6`
- `ATTENTION_AFTER_BLOCK = 3`
- `ATTENTION_HEADS = 4`
- `ATTENTION_HEAD_DIM = 16`
- `ATTENTION_DIM = 64`

The latest committed attention variant adds static geometry logits:

```text
 rel_bias[h, relation(i, j)]
+ dist_bias[h, distance(i, j)]
```

That experiment improved full validation and test BCE, but the learned bias table showed that the strongest signals were broad priors such as same-square suppression, near-square suppression, long-distance preference, and diagonal preference.

The new design removes those unconditional priors and tests whether the same relation and distance metadata are more useful as content-conditioned routing information.

## Agreed Architecture

Use full dense `64 x 64` board attention, but replace static relation/distance bias tables with zero-initialized edge gate vectors.

Base attention remains:

```text
base_logit[b, h, i, j]
  = q[b, h, i] @ k[b, h, j] / sqrt(16)
```

Edge gate contribution:

```text
edge_gate_logit[b, h, i, j]
  = scale * (
      sum_r relation_mask[r, i, j] * (
        q[b, h, i] @ rel_gate_q[h, r]
        + k[b, h, j] @ rel_gate_k[h, r]
      )
      + q[b, h, i] @ dist_gate_q[h, distance[i, j]]
      + k[b, h, j] @ dist_gate_k[h, distance[i, j]]
    )
```

Final logit:

```text
logit[b, h, i, j]
  = base_logit[b, h, i, j]
  + edge_gate_logit[b, h, i, j]
```

Where:

```text
scale = 16 ** -0.5
```

The edge gate is additive in logit space, but unlike static bias it depends on the current position through `q[b, h, i]` and `k[b, h, j]`.

## Parameter Contract

Remove these static bias parameters:

```text
rel_bias:  [heads, num_relations]
dist_bias: [heads, 8]
```

Add these zero-initialized gate parameters:

```text
rel_gate_q:  [heads, num_relations, head_dim] = [4, 9, 16]
rel_gate_k:  [heads, num_relations, head_dim] = [4, 9, 16]
dist_gate_q: [heads, 8, head_dim]             = [4, 8, 16]
dist_gate_k: [heads, 8, head_dim]             = [4, 8, 16]
```

Total edge-gate parameters:

```text
4 * 9 * 16 * 2 + 4 * 8 * 16 * 2 = 2,176
```

The parameter count increase over the static relation-bias model is:

```text
2,176 - 68 = 2,108
```

## Relation And Distance Contract

Keep the existing relation names and definitions:

| Relation | Definition |
| --- | --- |
| `same_square` | `dr == 0 and df == 0` |
| `same_rank` | `dr == 0 and df != 0` |
| `same_file` | `df == 0 and dr != 0` |
| `same_diagonal` | `dr == df and dr != 0` |
| `same_anti_diagonal` | `dr == -df and dr != 0` |
| `knight_move` | `{abs(dr), abs(df)} == {1, 2}` |
| `king_adjacent` | `max(abs(dr), abs(df)) == 1` |
| `our_pawn_attack_geometry` | `dr == 1 and abs(df) == 1` |
| `their_pawn_attack_geometry` | `dr == -1 and abs(df) == 1` |

The relation contract remains multi-label and additive. For example, a one-step diagonal pair may activate:

```text
same_diagonal + king_adjacent + our_pawn_attack_geometry
```

Use Chebyshev distance:

```text
distance[i, j] = max(abs(dr), abs(df))  # 0..7
```

Distance is still a separate signal from relation identity.

## Implementation Shape

Keep the implementation inside `NaiveBoardSelfAttention` for now to minimize model-callsite churn. The module behavior changes, but the external model input and output contracts remain unchanged.

Recommended buffers:

```text
relation_masks_flat: [9, 4096]
distance_index_flat: [4096]
distance_masks_flat: [8, 4096]
```

Use the existing flattening order:

```text
token_index = row * 8 + file
pair_index = query_token_index * 64 + key_token_index
```

Recommended forward shape:

```text
q_tokens: [batch, heads, 64, head_dim]
k_tokens: [batch, heads, 64, head_dim]
base_logits: [batch, heads, 64, 64]

edge_q_vectors: [heads, 64, 64, head_dim]
edge_k_vectors: [heads, 64, 64, head_dim]
```

Build `edge_q_vectors` and `edge_k_vectors` from relation masks, distance masks, and gate parameters:

```text
edge_q_vectors[h, i, j]
  = sum_r relation_mask[r, i, j] * rel_gate_q[h, r]
  + dist_gate_q[h, distance[i, j]]

edge_k_vectors[h, i, j]
  = sum_r relation_mask[r, i, j] * rel_gate_k[h, r]
  + dist_gate_k[h, distance[i, j]]
```

Then compute:

```text
edge_gate_logit[b, h, i, j]
  = scale * (
      q_tokens[b, h, i] @ edge_q_vectors[h, i, j]
      + k_tokens[b, h, j] @ edge_k_vectors[h, i, j]
    )
```

This vectorized form is mathematically equivalent to relation/distance score accumulation, but avoids a Python loop over relation and distance types during the main pairwise logit calculation.

Convert gate parameters and geometry masks to the active attention dtype inside `forward` so mixed precision training does not unexpectedly promote logits.

## Invariants And Boundaries

- Keep trunk width at `256`.
- Keep `4` heads and `16` dimensions per head.
- Keep the attention insertion point after block index `2`.
- Keep full dense `64 x 64` attention for this experiment.
- Keep relation and distance metadata board-fixed and deterministic.
- Remove all static relation and distance logit parameters.
- Initialize all edge gate parameters to exact zeros.
- With all edge gate parameters at zero, the module must be functionally equivalent to vanilla 4-head board attention.
- Do not add LayerNorm, dropout, FFN, sparse legal-move graphs, blocker-aware rays, or piece-conditioned edge schemas in this slice.
- Checkpoint compatibility with the static relation-bias model is intentionally not preserved.

## Test Plan

Add focused unit tests for:

- Edge gate parameter names, shapes, and exact zero initialization.
- Absence of static `rel_bias` and `dist_bias` parameters.
- Relation mask correctness for representative square pairs.
- Distance index and distance mask correctness for representative square pairs.
- Nonzero edge gate composition for a pair that activates multiple relations plus one distance bucket.
- Zero-gate equivalence against vanilla 4-head attention while Q/K/V/out projection weights are nonzero.
- Existing attention shape, finiteness, residual identity when all branch parameters are zeroed, and model placement tests.
- Explicit rejection of non-`8x8` attention inputs.

## Tradeoffs And Rejected Alternatives

- Keeping static `rel_bias` and `dist_bias` was rejected because this experiment should isolate content-conditioned edge gating without unconditional geometry priors.
- Sparse legal-move graph attention was rejected for this slice because legal move edges, blocker logic, and attack-map semantics would introduce multiple new variables at once.
- Full edge MLP over every `(query, key, relation, distance)` pair was rejected because it would be more expensive and harder to interpret.
- Relation-specific bilinear edge transforms were rejected for this slice because they multiply the attention dot-product cost by relation count.
- Renaming the module was deferred to avoid broad callsite churn during an experiment, but the design records that the module behavior changed.
