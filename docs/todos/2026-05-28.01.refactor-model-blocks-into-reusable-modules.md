## Context

python/libs/model.py contains model constants, material helpers, coordinate/ghost primitives, attention blocks, trunk assemblies, value heads, policy head, variant dispatch, and model factories in one large module. New experiments keep adding variants that reuse many of the same blocks.

## Why It Is Needed

The current single-file model surface makes it hard to add or compare variants without duplicating logic or touching unrelated behavior. Separating reusable primitives from variant assembly should reduce experiment risk while preserving checkpoint behavior.

## Completion Criteria

Model implementation is split into reusable modules with stable public imports, existing checkpoints still load, all model tests pass, parameter counts for existing variants remain unchanged, and no training/evaluation behavior changes are introduced intentionally.

## Detailed Plan

1. Introduce an internal python/libs/modeling/ package for model implementation pieces.
2. Move material helpers, primitive blocks, attention blocks, trunks, and heads into separate modules.
3. Keep python/libs/model.py as the stable compatibility facade exporting the existing public names.
4. Preserve existing ValueOnlyModel and ModelFull constructors and checkpoint state dict keys.
5. Run focused model tests and full unit tests after each slice.
