## Context

Material-label calibration analysis found that processed .chesseval files contain syntactically parsed but semantically impossible chess positions. In 1,000,000-row deterministic prefix samples, invalid piece geometry appeared in train, validation, and test at roughly 0.04%, including rows with impossible material differences such as -479..523 and examples like dozens of queens on one side. Follow-up DuckDB staging diagnosis found about 1.0% source rejects in 100k split samples, mostly invalid FENs that the old preprocess path already skipped, plus parse-valid nonstandard piece/pawn rows that need a strict standard-position filter. The Rust preprocess path now has a strict validator and rejection counters, but processed datasets still need regeneration and re-analysis.

## Why It Is Needed

Invalid board geometry corrupts material-difference features, fixed material-prior experiments, calibration analysis, and model training. Even if rare, extreme rows can create very large logits, gradient spikes, and misleading evaluation signals. Dataset regeneration should happen before further serious model experiments.

## Completion Criteria

Preprocessing rejects or quarantines non-standard positions with strict, documented constraints. Regenerated train/validation/test processed datasets have validity reports showing zero invalid piece geometry under the analyzer. The material-label calibration analysis is rerun on the regenerated data, and docs record the old-data contamination rate, new-data validity, and training implications.
