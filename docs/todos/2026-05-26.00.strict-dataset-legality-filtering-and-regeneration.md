## Context

Material-label calibration analysis found that processed .chesseval files contained syntactically parsed but semantically impossible chess positions. Follow-up DuckDB staging diagnosis found about 1.0% source rejects in 100k split samples, mostly invalid FENs that the old preprocess path already skipped, plus parse-valid nonstandard piece/pawn rows. The Rust preprocess path now enforces strict standard-position constraints and rejects extreme material positions with abs(material_diff) > 39. Processed train/validation/test datasets were regenerated from the rebuilt DuckDB staging database, and post-regeneration analyzer reports show zero invalid geometry for train 1M, validation full, and test full.

## Why It Is Needed

Invalid board geometry corrupts material-difference features, fixed material-prior experiments, calibration analysis, and model training. Even if rare, extreme rows can create very large logits, gradient spikes, and misleading evaluation signals. Dataset regeneration should happen before further serious model experiments.

## Completion Criteria

Preprocessing rejects or quarantines non-standard positions with strict, documented constraints. Regenerated train/validation/test processed datasets have validity reports showing zero invalid piece geometry under the analyzer. The material-label calibration analysis is rerun on the regenerated data, and docs record the old-data contamination rate, new-data validity, and training implications.
