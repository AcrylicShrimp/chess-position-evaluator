## Context

This TODO tracks cleanup work found during the initial project review on
2026-05-25. The project mixes Python model/training/evaluation code with Rust
preprocessing, ONNX inference, and benchmarking code. The review found several
artifact-compatibility, correctness, packaging, documentation, and repository
hygiene issues.

Priority guide:

- P0: Breaks a primary workflow or can produce clearly wrong chess/eval behavior.
- P1: Blocks maintainability, packaging, reproducibility, or common developer use.
- P2: Nice-to-have cleanup that reduces confusion or future drift.

## Why It Is Needed

The cleanup work needs a canonical tracker because the project had drift between
current code, committed model artifacts, README guidance, and utility scripts.
Some P0 issues have already been fixed, but follow-up P1/P2 items remain and
should be handled deliberately instead of rediscovered repeatedly.

### P1: Fix Battle Entrypoint Configuration

- [x] Remove hidden `BEST_CHECKPOINT_PATH` and stale default checkpoint
      selection.
- [x] Resolve explicit model names under `models/checkpoints`.
- [x] Route battle mode through the unified Typer CLI or document it clearly as
      a standalone script.

Implementation slice pseudo diff:

```diff
~ python/battle/entry.py
+ CHECKPOINTS_DIR = Path("models/checkpoints")
+ def resolve_checkpoint_path(model_name: str) -> Path:
+     if model_name is empty:
+         raise ValueError
+     return CHECKPOINTS_DIR / f"{model_name}.pth"
+ def run_battle(model_name: str) -> None:
+     load checkpoint from resolve_checkpoint_path(model_name)
+ def main() -> None:
+     parse required model_name argument
+     run_battle(model_name)

~ python/battle.py
+ import battle.entry.main as compatibility wrapper

~ python/cli.py
+ @app.command("battle")
+ def battle(model_name: str = typer.Argument(...)):
+     import battle entrypoint without conflicting with battle/ package
+     run_battle(model_name)

+ python/tests/test_battle_config.py
+ assert explicit model name maps to models/checkpoints/<model>.pth
+ assert empty model name is rejected
```

Evidence:

- `python/battle.py` checked `os.path.exists("BEST_CHECKPOINT_PATH")`, which
  tested a literal filename rather than the environment variable.
- The old default checkpoint path did not match the committed model directory.
- Checkpoint names vary by experiment, so battle now requires explicit model
  selection instead of a hidden default.

Acceptance checks:

- `python python/battle.py <model-name>` resolves
  `models/checkpoints/<model-name>.pth`.
- `cpe battle <model-name>` reaches the shared battle entrypoint.
- Empty model names are rejected before checkpoint loading.
